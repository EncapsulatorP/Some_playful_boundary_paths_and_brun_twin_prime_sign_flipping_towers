## 2. Core Mathematical Abstraction (to implement)

Define a reusable **boundary-residue framework**:

```python
L_∂(M_c, attractor_finder, anchor_b) → dist(A_c, b)
```

Where:
- `M_c` = iterative map (e.g., tower map `T_c(z) = c ** (c ** (-z))`)
- `A_c` = attracting object (fixed point `z+(c)` or cycle)
- `anchor_b` = reference boundary (1 for tower residue, 2 for Brun/Collatz Bay deficits)
- `L_∂` = boundary residue (signed or absolute distance)

**Key realizations in scope**:
- **Repunit Inverse Tower** (`repunit_tower.py`): `z+(c)`, `L_tower(c) = z+(c) - 1`, small-L series expansion, numerical fixed-point iteration.
- **Brun Bridge** (`brun_bridge.py`): `B_2` approximation + `L_Brun = 2 - B_2`; optional deeper twin-prime linkage to `prime-polarity`.
- **Collatz Bay** (`collatz_bay.py`): Proposed snap-back constant `c* ≈ 1.9086708647584145` and `L_Collatz = 2 - c*`.
- **General diagnostics** (`diagnostics.py`): `dL/dc`, fixed-point multiplier, local sensitivity, tunability solver (find `c` s.t. `L_tower(c) == L_Brun`).

**Stability bundle** (always computed alongside `L_∂`):
- Multiplier of the attracting fixed point/cycle
- Basin size / convergence rate indicators
- Lyapunov exponent (when relevant)

**Design principles** (align with your research values):
- Fully reproducible & script-driven
- Tunability explicitly quantified and warned (closeness is real but tunable)
- Diagnostic tool, **not** an identity claim
- Extensible map registry for future library growth
- Clear cross-links to RBOI, 0-ESG, CEC·CID, forced tetration, Kesten-Goldie, spectral methods, etc.
